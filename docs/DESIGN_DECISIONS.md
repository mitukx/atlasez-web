# デザイン決定（DESIGN_DECISIONS）

## 1. デザイン原則

- Wikipediaの機能的知見（固定目次・アンカー・情報階層）は取り入れるが、見た目は模倣しない
- 装飾を排し、色は「文字色・背景2種・罫線・アクセント1色」に限定
- 地図・冒険のメタファーは情報構造（学習地図・学習ルート・パンくず）にのみ反映し、装飾画像にはしない
- 巨大ヒーロー・グラデーション・ブロブ・無意味なアニメーションを使わない

## 2. 二つのサイトのトーン分離

共通のデザイントークン（`src/styles/tokens.css`）を基盤に、次で差別化する。

| 要素       | 公式サイト                   | 学習サイト                               |
| ---------- | ---------------------------- | ---------------------------------------- |
| アクセント | 紺 `#24425f`                 | 深緑 `#1e5c48`                           |
| ヘッダー   | 白地・下辺に太いアクセント線 | 薄灰地・細い罫線                         |
| 目的       | 活動と参加の理解             | 長時間の読書と探索                       |
| 本文幅     | 標準                         | `--reading-width: 42rem`（全角40字前後） |

`<html data-site="org|atlas">` でトークンを切り替える。

## 3. 分野ホームUI三案の評価と採用

実装比較は `/atlas/design-lab/subject-home/` を参照。

| 観点                 | 案A: 折り畳み           | 案B: プレビュー付きスクロール        | 案C: カード＋ミニ地図        |
| -------------------- | ----------------------- | ------------------------------------ | ---------------------------- |
| 初学者の理解しやすさ | ○ 大分類が一望できる    | ○ 記事が先に見える                   | ◎ 説明と入口が明示           |
| 記事数増加への拡張性 | ◎ 折り畳みで一定        | △ 横スクロールが伸び続ける           | ○ 「もっと見る」で吸収       |
| キーボード操作       | ◎ `<details>`は標準対応 | × 横スクロールの操作が困難           | ○                            |
| スマートフォン       | ◎ 縦積みで自然          | △ 横スクロール多用                   | ○                            |
| スクリーンリーダー   | ◎ 標準要素のみ          | △ スクロール領域の文脈が伝わりにくい | ○ ただしミニ地図の代替が必要 |
| 実装コスト           | ◎ JS不要                | ○                                    | △ グラフ描画が必要           |
| 情報密度             | ○                       | ◎                                    | ◎                            |

**採用: 案A（折り畳みセクション）を基調に、次の改良を加える。**

- 最初の大分類を初期展開し、クリック前から記事が見える（案Bの利点を取り込む）
- 各大分類に説明文・記事数・「記事一覧へ」リンクを表示（案Cの利点を取り込む）
- カテゴリ横断の俯瞰は学習地図ページが担う

実装は `src/pages/atlas/[locale]/[subject]/index.astro`。

## 4. 記事一覧

- グリッド（カード）／リスト（一行・高密度）の2表示。選択はlocalStorageに保存
- 演習未実装は「事前演習（準備中）」と明示（無効ボタンにしない）

## 5. 学習地図

- Obsidian風の「全ノード浮遊表示」を避け、目的（何をどの順に学ぶか）に集中
- エッジ種別は色＋線種＋矢印形状＋凡例テキストで区別（色覚多様性対応）
- 記事のないノードは白抜きで「準備中」を表現
- グラフはあくまで一表示手段。同一情報を階層リスト・表・経路フォームで提供

## 6. タイポグラフィ

- システムフォントスタック（外部フォント配信なし: プライバシー・速度・オフライン耐性）
- 本文 `line-height: 1.85`（日本語長文向け）。表示設定で2.2へ拡大可能
- 文字サイズはすべて `--text-scale` 経由で拡大に追従
