# 仮定の記録（ASSUMPTIONS）

仕様の曖昧点について置いた仮定。誤りがあれば修正指示をください。

1. **「基本HTMLサイト」の解釈**: 発注者確認済み。Astroで静的HTMLを生成する方式を採用（出力は純粋な静的サイトでGitHub Pagesにそのまま置ける）
2. **単一リポジトリ・単一ビルド**: MVPでは公式サイト（`/`）と学習サイト（`/atlas/`）を1つのAstroプロジェクトでビルド。GitHub Pages 1リポジトリで両方公開できるため。独自ドメイン取得後のサブドメイン分離手順はdocs/DEPLOYMENT.md参照
3. **パッケージ管理**: 仕様はpnpm推奨だが、検証環境でpnpmが利用できずnpmで検証（`package-lock.json`をコミット）。pnpm利用時は `pnpm import && pnpm install` で移行可能
4. **学習地図の実装**: Preact IslandではなくAstroのバンドルスクリプト（vanilla TS + Cytoscape.js）で実装。UIフレームワーク依存を避けるため（ADR-001に記録）
5. **ロゴ画像**: 現行サイトの画像はGoogleの配信URLからの自動取得が本環境で不可だったため、差し替え前提の簡素なSVG仮ロゴを作成。取得元URLと差し替え手順はdocs/MIGRATION_PLAN.md §3
6. **公式サイトの文章**: 2026年7月23日に現行Google Sitesから「Atlasezとは」「理念・目標」「組織構成」を移植。沿革は現行サイト自体が準備中のため、内容を推測せず準備中表示とする
7. **運営メンバー**: 現行Google Sitesで一般公開されている氏名・所属・担当・自己紹介を移植し、公式サイトと学習サイトで同じデータを参照する
8. **お問い合わせフォーム**: スパム対策（reCAPTCHA等の外部スクリプト）とプライバシー方針の兼ね合いから、MVPではメール・SNS・GitHub Issueでの受付とした
9. **英語UIの範囲**: 学習サイトはja/en対応。公式サイトは当面日本語のみ（現行サイトも日本語のみのため）
10. **化学の他言語版**（韓国語・中文・台湾華語・タイ語）: ロケール追加は設計上可能だが、MVPではja/enのみ実装
11. **演習**: `exerciseIds` フィールドと「準備中」表示のみ実装（プレースホルダー）。演習コンテンツ自体は未実装
12. **GitHubリポジトリURL**: 公開先として指定された `https://github.com/mitukx/atlasez-web` を使用
13. **Lighthouse検査**: CIに組み込み済みだが、初回は警告のみ（`|| true`）。基準値の強制は運用開始後に調整
14. **E2Eテスト**: Playwright（ブラウザ実行）はサンドボックス制約のためローカル未実行。CIで実行される。単体テスト・ビルド・リンク検証・スキーマ検証はローカルで実行済み
15. **抽出JSONの対象言語**: `atlas-articles-2.json` の生物分野に含まれるタイ語ページは、MVPの対応ロケール（ja/en）外であるため今回の日本語記事移植から除外。ロケール追加時に別記事として移植する
16. **既存概念IDの再利用**: 数学・化学の抽出記事が既存の概念を扱う場合は、新しい概念IDを作らず公開済みの永続IDを再利用する。このため概念数は抽出ページ数との単純合計にはならない
17. **追加分野のグラフ接続**: 物理・生物・古文は本文の依存関係に基づき `prerequisites` を設定。漢字のテーマ別記事は相互に必須とは言い切れないため、カテゴリ内の掲載順を `recommendedNext` として接続する。単独カテゴリの記事は内容上近い概念を `related` で接続する
18. **本文未完成の記事**: 現行Google Sitesの目次に題名だけがある項目は、内容や概念を推測して記事ページを生成せず、`src/data/planned-articles.json` で管理して分野・カテゴリ一覧に「準備中」と表示する。本文が移植された時点で同データから削除し、通常の記事コレクションへ追加する
