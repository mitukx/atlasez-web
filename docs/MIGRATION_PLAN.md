# 移行計画（MIGRATION_PLAN）

現行Google Sitesは移行完了まで停止しない。段階移行を前提とする。

## 1. 段階移行の手順

| 段階                  | 内容                                                             | 状態    |
| --------------------- | ---------------------------------------------------------------- | ------- |
| 1. 新サイト試作       | 本リポジトリ（構造・機能・サンプル記事）                         | ✅ 完了 |
| 2. 内部レビュー       | 運営メンバーでUI・情報設計・文言を確認                           | 次      |
| 3. 一部分野の移行     | 数学・化学の全記事をMarkdown化しPR/査読                          |         |
| 4. ベータ公開         | GitHub Pagesで公開、現行サイトに「新サイト（ベータ）」リンク掲載 |         |
| 5. フィードバック収集 | GitHub Issue・お問い合わせ・SNSで収集し反映                      |         |
| 6. 全体移行           | 残り分野・公式サイト文章・画像の移行                             |         |
| 7. 移転案内           | 各Google Sitesの先頭に新URLへの案内バナーを掲載                  |         |
| 8. アーカイブ         | 案内掲載から6か月後を目安に旧サイトを非公開化（編集用に保管）    |         |

## 2. URL移行表

`docs/CURRENT_SITE_AUDIT.md` §7の表を正とする。移行の進行はこの表に「担当者・ステータス」列を追記して管理する。

| 列             | 値                                                                             |
| -------------- | ------------------------------------------------------------------------------ |
| 移行ステータス | 未着手 / 移行中 / 査読中 / 公開済み                                            |
| 優先度         | 高（公式主要ページ・数学・化学）/ 中（他公開分野）/ 低（未公開分野・多言語版） |
| 担当者         | GitHubユーザー名                                                               |

## 3. コンテンツごとの方針

- **公式サイトの文章**（理念・組織・沿革・お知らせ過去記事）: 現行サイトからコピーし、`src/pages/about/*` のプレースホルダー文言（「移行用の下書き」と明記済み）を置き換える。お知らせ過去記事は `src/content/news/` にMarkdownで追加
- **学習記事**: Google Sitesの各ページをMarkdown化。①本文をコピー → ②数式をKaTeX記法へ → ③frontmatter付与（概念ID割当・前提関係の定義）→ ④PR・査読。**この工程で初めて前提関係グラフが構築されるため、分野担当者が概念表を先に作ること**
- **未完成記事**（現行サイトのリンクなし項目）: 概念だけ `concepts.yaml` に登録し、記事なし＝「準備中」として表示（実装済みの挙動）
- **画像・添付ファイル**: 現行サイトから元データを収集し `public/images/<subject>/` に配置。出典・ライセンスを確認できないものは移行しない
- **ロゴ**: 正式ロゴ画像を `public/images/atlasez-logo.svg`（または.png）と `public/images/atlas-logo.svg` に差し替える。現行の取得元:
  - 公式ロゴ: Google Sites `atlasez` のヘッダー画像（サイト管理画面からオリジナルを取得）
  - 学習サイトロゴ: `learning-atlas` のヘッダー画像
  - ※ 本リポジトリの仮SVGロゴは差し替え前提のプレースホルダー
- **多言語版（化学の韓国語・中文・台湾華語・タイ語）**: ロケール追加（`src/lib/i18n.ts` の `LOCALES` と `content.config.ts`）後に移行。優先度低

## 4. 旧URLからの案内

- Google Sitesはサーバーリダイレクトが設定できないため、各ページ先頭に新URLへのリンクバナーを手動掲載する
- 新サイトの404ページには主要導線を設置済み
- SNSプロフィール・外部リンクを新URLへ順次更新

## 5. ロールバック

- 新サイトはGitで全履歴が残るため、問題があれば直前のコミットへrevertして再デプロイ
- ベータ期間中は現行Google Sitesが常に利用可能（案内バナーのみ撤去すれば完全に元の状態）
